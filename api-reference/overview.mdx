A API da **Sync Pay** foi projetada para ser simples, rápida e segura. Baseada em padrões REST, ela permite que desenvolvedores integrem fluxos complexos de pagamentos e recebimentos com poucas linhas de código.

## Endpoints de Produção

Todas as requisições devem ser direcionadas para o nosso servidor de produção:

<Card title="Base URL" icon="link">
  `https://api.syncpay.com.br`
</Card>

<Warning>
  Sempre utilize HTTPS para garantir que os dados trafegados entre sua aplicação e nossos servidores estejam criptografados.
</Warning>

## Autenticação Segura

Para interagir com nossa API, você precisa de uma **API Key**. Esta chave é pessoal e intransferível.

### Como autenticar:

Adicione sua chave ao header de cada requisição utilizando um dos métodos abaixo:

**Método Recomendado (Header Customizado):**
```http
X-API-Key: sua_api_key_aqui
```

**Método Padrão (Bearer Token):**
```http
Authorization: Bearer sua_api_key_aqui
```

---

## Padrões de Resposta

Nossa API utiliza o formato JSON para todas as respostas. Cada resposta contém um booleano `success` para facilitar o tratamento de erros no seu código.

### Exemplo de Sucesso (200 OK)
```json
{
  "success": true,
  "data": {
    "id": "pay_123456",
    "status": "PENDING",
    "amount": 150.00
  }
}
```

### Exemplo de Erro (400 Bad Request)
```json
{
  "success": false,
  "error": "INVALID_AMOUNT",
  "message": "O valor do pagamento deve ser maior que R$ 1,00"
}
```
